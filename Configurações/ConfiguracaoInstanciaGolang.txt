 1.8) Rode com o comando ./app
 1.9) mas se quiser deixar rodando em backgroup e poder mexer no terminal digitar nohup ./app > app.log 2>&1 &
 2.0) para verificar se esta rodando-> ps aux | grep ./app isso gera um novo processo, se não quiser isso e quiser saber qual numero matar digitar o seguinte

 3) Para deixar a aplicação golang rodando automaticamente
  3.1) sudo nano /etc/systemd/system/flyturaGolang-app.service
  3.1.1) conteudo do arquivo

       [Unit]
       Description=Minha aplicação Go
       After=network.target

      [Service]
      ExecStart=/home/ec2-user/app
      WorkingDirectory=/home/ec2-user
      Restart=always
      RestartSec=5
      User=ec2-user
      Environment=PORT=8080

      [Install]

  3.2) Ativar o serviço
    3.2.1) sudo systemctl daemon-reload
    3.2.2) sudo systemctl enable flyturaGolang-app.service
    3.2.3) sudo systemctl start flyturaGolang-app.service
  3.3) Verificar se está rodando
    3.3.1) sudo systemctl status flyturaGolang-app.service
  3.4) ver logs da aplicação
    3.4.1) journalctl -u flyturaGolang-app.service -f
  3.5) para ou reiniciar se necessário
    3.5.1) sudo systemctl stop flyturaGolang-app.service -f
    3.5.2) sudo systemctl restart flyturaGolang-app.service

  3.6) para remover um serviço
    3.6.1) sudo rm /etc/systemd/system/flyturaGolang-app.service
    3.6.2) para atualizar o systemd -> sudo systemctl daemon-reload
